Acho que pode ser interessente deixar ir registrando as etapas que eu segui para criar esse site. Isso me ajuda a sistematizar melhor o conhecimento que adquiri até o momento e também pode me ajudar no futuro, quando eu for tentar aprender coisas mais complexas.

Esse blog é um site estático. Isso significa que é composto por arquivos fixos (normalmente HTML, CSS e às vezes JavaScript) que são enviados para o navegador exatamente como estão armazenados no servidor. Tudo o que eu escrevi/escrevo nos arquivos HTML e CSS que compõem esse blog (não incluí JavaScript, ainda) é exatamente o que a pessoa que está acessando recebe.

Primeiro eu criei os arquivos no meu computador, depois coloquei em uma pasta criada no meu perfil no GitHub. É muito simples fazer isso utilizando o recurso GitHub Pages, que hospeda o site gratuitamente com o um endereço do tipo usuario.github.io.

Etapas de publicação

1. Desenvolvimento local

Minha primeira ideia foi pegar uma referência de site que eu achava interessante e tentar reproduzir. Acabei me deparando com a possiblidade de utilizar a ferramenta Jekyll para facilitar a vida e deixar tudo mais bonito. Mas pensei que isso poderia acabar não sendo muito diferente de criar um blog no Wordpress, coisa que eu já fiz antes. Então optei pelo caminho de abrir uma tela em branco e ir escrevendo cada linha.

Criei um arquivo html com o nome index.html. Ele funciona como a porta de entrada do site, a home, onde coloquei uma breve apresentação e os links de cada blogpost. Depois, criei o arquivo style.css, para que o site pudesse ficar mais estilizado. Foi quando surgiu a ideia de misturar a ideia de um blog pessoal de estudos com conhecimentos aleatórios sobre focas.

Resolvi criar uma página para cada blogpost. Aqui passei pela minha primeira dificuldade, pois não sabia como conectar a página principal com as páginas de dentro da pasta 'posts'. Localmente isso estava funcionando perfeitamente, mas quando fiz as modificações no meu repositório do GitHub, a página clicada não era encontrada. Acabei descobrindo que esse problema era parecido com esse aqui. Nessa brincadeira, acabei aprendendo um pouco mais sobre o erro 404, mas deixarei isso pra depois.

Quando o site estava publicado no github e eu já conseguia acessar usando qualquer dispositivo, descobri que para que ele ficasse responsivo, isso é, se adaptasse para telas menores como celular, eu precisaria fazer mais algumas modificações. Teria que utilizar unidades flexiveis no CSS, tornar os tamanhos de imagens flexíveis e implementar Media Queries, que faz com que sejam aplicados estilos diferentes dependendo da largura da tela do usuário.

Exemplo:

body { font-size: 16px; } @media (max-width: 768px) { body { font-size: 14px; } nav { display: none;}}

Nesse caso, tudo que for incluído no 'body' depois de @media só será aplicado quando o tamanho da tela for menor que 768px.
